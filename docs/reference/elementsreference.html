Elements Reference

Transformable Elements

GLAM transformable elements are elements that can be positioned, rotated and scaled, including
  • <group>
  • <import>
  • <camera>
  • <light>
and the visual elements
  • <box>
  • <cone>
  • <cylinder>
  • <sphere>
  • <text>
  • <mesh>
  • <points>
  • <lines>
  • <particles>
  • <arc>
  • <circle>
  • <rect>
Transformable elements may be transformed using the CSS transform property, or by setting the position, rotation and scale attributes listed below.

Attributes

[attribute:x]

The x coordinate of the element's position. This is a synonym for px.

[attribute:y]

The y coordinate of the element's position. This is a synonym for py.

[attribute:z]

The z coordinate of the element's position. This is a synonym for pz.

[attribute:rx]

The x coordinate of the element's rotation.

[attribute:ry]

The y coordinate of the element's rotation.

[attribute:rz]

The z coordinate of the element's rotation.

[attribute:sx]

The x coordinate of the element's scale.

[attribute:sy]

The y coordinate of the element's scale.

[attribute:sz]

The z coordinate of the element's scale.

[attribute:px]

The x coordinate of the element's position.

[attribute:py]

The y coordinate of the element's position.

[attribute:pz]

The z coordinate of the element's position.

Visual Elements

Visual elements are elements that have a visible appearance in a GLAM scene, including
  • <box>
  • <cone>
  • <cylinder>
  • <sphere>
  • <text>
  • <mesh>
  • <points>
  • <lines>
  • <particles>
  • <arc>
  • <circle>
  • <rect>
The appearance of visual elements may be changed using CSS material properties such as diffuse-color and image.

The 'glam' element

The root element of a glam document.

Creating

<glam>

This creates the root GLAM document. Under the covers, this instantiates a GLAM application object that manages Three.js renderer and scene creation, window sizing and other events, as well as the requestAnimationFrame run loop. In the current implementation, the root <glam> tag must be created as a child of a <div> element. This restriction will probably go away soon.

Attributes

This element has no attributes.

The 'scene' element

The root element of the visual scene. All visual objects must be created as child elements of a <scene> element.

Creating

<scene>

This creates the root object of the visual scene. In the current implementation, there can be only one <scene> element per GLAM document. This restriction may go away soon.

Attributes

This element has no attributes.

The 'group' element

A grouping element used to create the scene graph hierarchy.
Groups are transformable elements; their position, rotation and scale can be changed. Transforms defined for a 'group' element affect all of its descendants. 'group' elements can be nested, thus creating a hierarchy of objects and their transformations. (For more information, see The Transformation Hierarchy.)

Non-transformable elements such as 'animation' or 'effect' can also be grouped (by creating them as children of 'group' elements) for convenience, but the transformation hierarchy has no effect on those elements.

Creating

<group>

Creates a group. A group can contain zero or more GLAM elements, either leaf elements (visuals, lights, cameras etc) or other groups.

Attributes

Groups inherit the position, rotation and scale attributes defined for transformable elements.
