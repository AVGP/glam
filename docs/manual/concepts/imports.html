Importing Models

GLAM provides authors a way to embed external 3D data into the document. This data is stored in other standard 3D formats. This is analogous to embedding image files in JPEG or PNG format into a 2D HTML document. GLAM is not designed for compact representation, but rather for ease of authoring and interactivity. Therefore, as a practical matter it will often be desirable to embed models in formats intended to solve those problems, such as OBJ, COLLADA and glTF. GLAM achieves this through use of imports.

The following example shows a GLAM scene that imports an OBJ model and adds a camera. The import element's src property specifies a URI to the imported scene file.

<glam> <scene> <import id="imported_scene" src="./models/model.obj"> </import> <camera></camera> </scene> </glam>

A GLAM scene can contain any number of 'import' elements. This is useful for importing multiple models into the same scene. 'import' elements can be transformed (translated, rotated, scaled) and animated just like any other GLAM element.
